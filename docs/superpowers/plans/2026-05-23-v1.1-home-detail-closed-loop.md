# v1.1 Home + Detail Closed-Loop Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the v1.1 keep/kill closed loop — Home becomes a monthly triage console with an annual-spend hero, sectioned list, and per-row relative renewal; Detail becomes a single-subscription decision page with a prominent countdown, color-coded price history, and a tappable URL. FX rework moves conversion from "pinned at creation" to "converted at display time" via a cached `FXRateTable`.

**Architecture:** All work renders in the existing pixel-art system (`TrackrColors`, `PixelText`, VT323). Pure helpers (`MonthlyTotalCalculator`, `AnnualSpendCalculator`, `RelativeRenewalText`, `HomeSectionBuilder`, `SpendAnchorCatalog`, `SpendAnchorRenderer`, `PriceHistoryFormatter`) keep view code thin and unit-testable without a `ModelContext`. SwiftData model additions are strictly additive (`PriceHistoryEntry`, `FXRateTable`, `Subscription.trialEndsAt`) so existing TestFlight rows migrate without a destructive migration.

**Tech Stack:** Swift 5, SwiftUI, SwiftData, swift-snapshot-testing, XCTest. iOS 17+ deployment target. Frankfurter API for FX refresh, bundled `fx-fallback.json` for first launch.

## Migration safety

- `Subscription.exchangeRateToHome` / `homeCurrencyAtCreation` /
  `exchangeRateAsOf` stay on the model — legacy TestFlight rows read
  cleanly but are ignored by aggregation.
- `Subscription.trialEndsAt: Date?` is additive (nullable, default nil).
- `PriceHistoryEntry` and `FXRateTable` are new `@Model`s registered in
  `ModelContainerConfig.schema`.
- First-launch backfill writes one `.initial` row per legacy sub,
  idempotently — re-launching never duplicates.

## What didn't change

- Pixel-art design system, `TrackrColors`, `PixelText`, VT323.
- PAUSE / DELETE remain the only state actions; no CANCELLED state
  added (Open Question 1 resolution in design doc).
- `PriceChangeAlert` + `AlertRepository` untouched — alerting is
  orthogonal to history.
- Calendar screen still deferred to ROADMAP Phase 2.
